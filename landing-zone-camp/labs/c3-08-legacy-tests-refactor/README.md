# c3-08 遗留代码：先补测试再重构             ⏱ 45min | 能力域 7 | 判题 CI

## 背景
对没有测试的遗留函数，重构前先写**表征测试**（characterization tests）：把现状（包括怪行为）钉死，再放心让 AI 重构。

## 任务
`starter/date_parse.py` 是一个无测试的老函数，带两个"怪但被下游依赖"的行为（读代码找出来）。
1. 先补 `starter/test_date_parse.py`：≥4 个测试，其中 ≥2 个钉住怪行为（函数名含 `quirk`），全绿
2. 再让 AI 重构 `parse_date` 内部实现（可读性/结构），**函数签名与全部测试不许动**
3. `refactor-note.md` 记录：两处怪行为是什么 / 为什么重构时保留它们

## 判题
- 🟡 CI：测试 ≥4（含 ≥2 个 quirk）全绿 + 签名未变 + note 两节齐全
- 🟢 教练抽查：如果业务方现在要"修掉"怪行为，该走哪级 Spec？（答：L1，行为变更）
