# c3-10 需求单到发布单全链路演练             ⏱ 45min | 能力域 13 | 判题 CI

## 背景
编码只占研发全流程约 30%。C3 收官：把一个需求从"需求单"推到"发布单"，每一站都有 AI 参与、都有产物、都互相引用。

## 任务
沿用 c3-09 的"限用一次"需求，产出 `pipeline.md`，八个检查项全部完成并勾选，且每项附产物链接/路径：
- [ ] 需求单（含 Spec 级别判定）
- [ ] L2 Spec（引用 c2-02 模板）
- [ ] Spec 评审记录
- [ ] 实现 MR 链接
- [ ] 测试报告（AC 覆盖表）
- [ ] 四维 Review 记录
- [ ] 灰度/回滚预案（各 ≥2 行）
- [ ] 发布单（含监控指标 ≥2 个）

## 判题
- 🟡 CI：8 项全部为 `[x]` 且每项后有链接/路径；回滚与监控关键词在文中出现
- 🟢 教练答辩：随机抽一站问"这一站 AI 干了什么、人把了什么关"
