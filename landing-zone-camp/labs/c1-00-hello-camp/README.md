# c1-00 hello, camp：环境准备与第一次提交    ⏱ 15min | 能力域 1 | 判题 CI

## 背景
这一关的目的只有一个：**10 分钟走通「改文件 → 推分支 → 自动判题 → 平台亮灯」的完整链路**。走通这一关，之后 30 关全部复用同一条链路，只是改的文件不同。如果你已经会用 git 改文件、推分支，这关很快；如果不会，跟着下面的命令一步步来。

本系统采用**共享训练仓库 + 个人分支**模型：所有学员 clone 同一个训练仓库（不是各自 fork），在自己分支上提交。CI 在这个共享仓库上跑判题并回调平台。

## 任务

按下面的步骤做。命令里的 `<工号>`、`<姓名>`、`<训练仓库地址>` 请替换成你自己的（仓库地址见平台「新手引导」页）。

**第 1 步：拿到仓库访问权限**
找营长或组长开通训练仓库的访问权限（push 权限）。仓库地址在平台「新手引导」页。

**第 2 步：克隆仓库到本地**
```sh
git clone <训练仓库地址>
cd <仓库目录>          # 即 landing-zone-camp 所在的仓库根目录
```

**第 3 步：配置你的 git 身份**（每个仓库只需配一次）
```sh
git config user.name "你的姓名"
git config user.email "<工号>@公司邮箱域"
```

**第 4 步：建分支**
```sh
git checkout -b camp/c1-00/<工号>
```
> ⚠️ **分支名里的工号是平台认人的唯一依据，写错 = 白交。** 格式严格是 `camp/c1-00/你的工号`。

**第 5 步：进入本关目录，创建 hello.md**
```sh
cd labs/c1-00-hello-camp
```
在本目录创建 `hello.md`，内容三行：
```
工号: <你的工号>
姓名: <你的姓名>
我最想让 AI 帮我解决的事: <一句话>
```

**第 6 步：提交并推送**
```sh
git add .
git commit -m "chore(camp): c1-00 hello"
git push -u origin camp/c1-00/<工号>
```

**第 7 步：回平台看结果**
回到平台刷新本关页面。约 1 分钟内，「判分记录」会出现 CI 事件、状态变 🟡 —— **这个闭环就是你之后每一关的提交方式**。

## 判题
- 🟡 CI：`hello.md` 三要素齐全（工号、姓名各一行且冒号后非空，外加至少一行有效内容）
- 🟢 教练在开营周集中复核（确认其真人真号对得上）

## 你会用到
平台「新手引导」页（有已替换好你工号的完整命令，可一键复制）· [handbook/grading.md](../../handbook/grading.md)（判题机制说明）

## 提交方式
分支 `camp/c1-00/<工号>`，推送后 CI 自动判题。
