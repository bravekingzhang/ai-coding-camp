# 功能 Spec：消息通知（作者：赶时间的同事）

## 背景
用户希望及时知道订单状态变化，所以要做通知。

## 用户故事
作为用户，我希望收到通知，以便了解订单。

## 验收条件
### 正常流程
- [ ] WHEN 订单创建 THEN 尽快给用户发通知，体验要好          <!-- 缺陷1：不可测 -->
- [ ] WHEN 支付成功 THEN 通知内容友好、准确                  <!-- 缺陷2：不可测 -->
- [ ] WHEN 订单超时取消 THEN 发送取消通知，包含订单号
### 异常流程
（暂无，正常情况覆盖即可）                                    <!-- 缺陷3：第三方通道失败未定义 -->
                                                              <!-- 缺陷4：用户已关闭该类通知未定义 -->
### 边界条件
- [ ] WHEN 用户手机号为空 THEN 跳过发送

## 非功能需求
| 维度 | 要求 |
|------|------|
| 性能 | 通知要快 |
| 安全 | 常规即可 |
| 幂等 |            |                                        <!-- 缺陷5：重复触发重复发，无幂等键 -->

## 依赖与影响范围
- 改动的文件：通知相关的几个吧                                <!-- 缺陷7：影响范围含糊，未提新表 notify_log -->
- 是否涉及 DB 变更：应该不涉及
- 是否涉及 API 变更：新增 POST /notify，返回 {"ok": true}     <!-- 缺陷8：违反 code/message/data 返回体约定 -->
