# c2-02 写一份 L2 标准 Spec                ⏱ 45min | 能力域 3 | 判题 HYBRID | 提交后自动指派跨组评审

## 背景
Spec 是团队协作的"接口格式"。本关产物将**自动指派给另一个小组的同学评审**（你也会收到别人的）。

## 任务
需求："给订单系统新增消息通知功能：订单创建、支付成功、超时取消三个时点通知用户，用户可关闭某类通知。"
按 [templates/l2-standard-spec.md](../../templates/l2-standard-spec.md) 写 `l2-spec.md`，必须包含：
背景 / 用户故事 / 验收条件（正常流程、异常流程、边界条件三小节，全部 EARS）/ 非功能需求（性能、安全、幂等）/ 依赖与影响范围。

## 判题
- 🟡 CI 结构检查 → LLM rubric（rubrics/spec-quality.yaml，10 项）预评 ≥7 分
- 🟢 教练复核，且跨组评审无 🔴 结论

## 提交方式
分支 `camp/c2-02/<工号>`（推送即触发评审指派）
