# c3-01 解剖一个好 Skill                   ⏱ 25min | 能力域 10 | 判题 CI

## 背景
Skill 是把踩坑经验工程化为可复用文档的载体（什么是 Skill 见 [handbook/skill-guide.md](../../handbook/skill-guide.md)）。会写之前先会读：一个好 Skill 的每个部分都有存在理由。

## 任务
精读 `starter/sample-skill/SKILL.md`（MySQL 慢查询排查，标准六段结构）。
写 `anatomy.md`，六个小节固定为：`## frontmatter`、`## 触发时机`、`## 步骤`、`## 反例`、`## 边界`、`## 维护信息`——
每节回答两问：这部分给谁看（人/AI）？删掉它会发生什么？

## 判题
- 🟡 CI：六节齐全且每节 ≥2 行
- 🟢 教练抽查：指出 sample 里"最容易被新手省略但最值钱"的一节并说明理由
