# M15 效能度量（C4 · 工程教练）            ⏱ 3h（含实操）

## 学习目标
能独立为一个部门建立六维效能基线（Adoption / Engagement / Acceptance / Delivery Flow / Quality / Cost），并写出管理层看得懂的月报。

## 内容
1. 六维指标口径逐条拆解：每个指标"用没用/用多深/好不好/快没快/稳不稳/花多少"分别回答什么问题、数据从哪来（AI 工具后台统计 + Git 平台提交统计）
2. 基线采集方案：先采 4 周静默基线再开营，避免"边改边量"
3. 智能归因三件套：超大提交过滤、Merge 排除、Squash/Amend 去重
4. ROI 叙事：为什么 12 周先看能力与资产指标，交付类指标滞后 1~2 个季度 —— 预期管理话术

## 实操作业（计入 P3）
用本部门近 4 周真实数据填 `baseline-template.md`，产出首份基线报告并在周五分享会讲 10 分钟。

## 通过标准
🟡 基线报告数据可追溯、口径写明；🟢 报告被营长采用为 W12 对比基准
