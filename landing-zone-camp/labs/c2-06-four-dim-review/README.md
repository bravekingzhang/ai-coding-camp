# c2-06 四维审查实战                       ⏱ 40min | 能力域 6 | 判题 CI

## 背景
Code Review 四级法把审查拆成四个正交维度：**安全 / 正确性 / 性能 / 可维护性**。`starter/user_service.py` 每个维度各埋了 1 处问题。

## 任务
写 `findings.md`，四个小节标题固定为：`## 安全`、`## 正确性`、`## 性能`、`## 可维护性`。
每节至少一条：`行号 | 问题 | 后果 | 修法`。
提示：安全看输入如何进 SQL；正确性用一页数据算一遍边界；性能数一数循环里的查询次数；可维护性找"看不懂为什么"的数字。

## 判题
- 🟡 CI：四节齐全且每维度命中关键点（SQL 注入 / 分页边界 / N+1 / 魔法数）
- 🟢 教练抽查：给出每个问题"AI 审得出来吗"的判断——哪些必须人看
