# landing-zone-camp · 训练内容包（课程即代码）

平台从本目录读取一切课程内容：**改课程 = 改本仓库 + 平台点"同步"**。

```
curriculum.yaml     关卡地图（30 关 · 解锁依赖 · 判题类型）—— 平台同步的唯一来源
labs/               30 个训练题目：README 任务卡 + starter 材料 + .verify 判题
projects/           P1~P3 认证项目卡 + S1~S4 团队场景卡
c4-modules/         C4 教练四模块（M15~M18）
rubrics/            LLM 预评评分细则 + 教练答案基准（coach-keys.md 开营前移入私有仓库！）
placement/          入营摸底三题（分层 + test-out 判定）
ci-templates/       训练仓库用的 .gitlab-ci.yml 与回调脚本
tools/              共享判题工具（答案哈希比对等）
```

## 学员使用方式
1. **从 c1-00 开始**：它是「环境准备 + 第一次提交」的引导关，走通"改文件→推分支→自动判题→平台亮灯"的完整链路（约 15 分钟）。之后每一关都用同一套提交方式。
2. clone 训练仓库（labs 目录随仓库分发；仓库地址见平台「新手引导」页）
3. 做题：产物按各关 README 放置 → 建分支 `camp/<关卡号>/<工号>` 推送
4. CI 自动跑该关 `.verify/verify.sh` 并回传平台 → 🔴/🟡 即时可见；🟢 由教练/互测判定
5. 本地预跑：任意关卡目录下 `sh .verify/verify.sh`

## 判题设计三原则
1. 🟡 条件全部客观可测（脚本/哈希/结构检查），🟢 永远需要人或互测 —— 与颜色归约器规则一致
2. 测验类答案只存哈希（tools/check-answers.sh），埋雷类材料的基准在 rubrics/coach-keys.md
3. .verify 里跑稳的检查项在 W6/W12 迁入生产门禁 —— 判题即门禁

## 出新题模板
复制任意关卡目录，README 六节（标题行/背景/任务/判题/你会用到/提交方式），verify 保证「学员没做对时给出**指路**而非仅报错」。
